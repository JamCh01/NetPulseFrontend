# NetPulse 产品需求文档（PRD v1.1）

Last updated: 2026-05-19
Version: v1.1
Owner: NetPulse Frontend
Status: Draft for product positioning and UX optimization

## 1. 背景与问题

NetPulse 当前已具备较完整的监控与运维能力闭环（任务、Agent、监控、告警、Webhook、管理后台），但在产品层面仍存在三个关键问题：

1. 差异化表达不够尖锐：功能很多，但“为什么必须选 NetPulse”不够一眼可知。
2. 价值路径不够显性：从发现异常到定位根因再到触发处置的闭环，交互上仍偏分散。
3. 结果呈现不够业务化：对管理者/客户层缺少可直接消费的价值产出（如 SLA 与复盘输出）。

## 2. v1.1 目标

### 2.1 产品定位目标

将 NetPulse 从“监控功能集合”升级为“链路诊断与自动处置平台”，形成清晰主张：

- 多地域真实链路视角
- MTR 级根因定位
- 自动联动处置能力

### 2.2 体验目标

- 将核心闭环交互化：发现 -> 定位 -> 处置
- 降低关键操作路径长度（减少跨页与手动筛选）
- 提升用户对“监控价值”的即时感知与复用能力

## 3. 目标用户（v1.1 关注）

1. 运维工程师（Primary）
- 诉求：快速发现异常、快速定位原因、快速推进处置。

2. 值班/应急响应人员（Primary）
- 诉求：在告警发生时，最短路径拿到“可行动信息”。

3. 技术管理者（Secondary）
- 诉求：关注稳定性趋势、SLA、复盘结果与团队效率。

## 4. 核心竞争力（v1.1 定义）

## 4.1 多视角定位能力

不仅看“目标是否可达”，还可从多 Agent 视角对比异常范围，判断是目标问题、区域问题还是链路问题。

## 4.2 诊断深度能力

结合 MTR 结果形成“链路层证据”，帮助从症状走向根因，而不是停留在告警噪音。

## 4.3 处置联动能力

通过规则与 Webhook 将“发现问题”转化为“自动触发协同流程”，缩短响应链路。

## 5. 需求价值

## 5.1 业务价值

- 降低故障发现和定位时延（MTTD/MTTR）
- 提高值班效率，减少重复人工判断
- 形成可复用的故障处理与复盘资产

## 5.2 用户价值

- 工程师：更少页面切换，更快得到可执行判断
- 值班人员：告警即上下文，不再到处拼信息
- 管理者：可直接消费的稳定性与故障治理输出

## 6. v1.1 关键需求（新增/强化）

## 6.1 信息传达层（Positioning & IA）

1. 首屏价值锚点（3段式）
- 监控范围（监控什么）
- 诊断深度（如何更快定位）
- 自动处置（如何更快响应）

2. 模块命名结果化
- Dashboard -> 全网健康
- Monitoring -> 链路诊断
- Alerts -> 异常处置
- Webhooks -> 自动联动

3. 每页增加“价值说明条”
- 用一行语言说明当前页面对用户的直接价值与决策意义。

## 6.2 交互层（核心闭环优化）

1. 异常优先视图（P0）
- 列表默认优先展示异常任务/区域
- 一键切换“仅看异常”

2. 告警到诊断一跳直达（P0）
- 从告警事件直接进入对应任务+时间窗+MTR上下文
- 降低手动筛选成本

3. 监控详情“结论先行”结构（P1）
- 顶部固定结论卡：当前状态、影响范围、疑似根因
- 图表与明细作为证据层

4. 告警规则模板化（P1）
- 预置高延迟/高丢包/抖动异常模板
- 降低规则配置门槛

5. 运营化输出（P2）
- SLA 周报导出
- 故障复盘摘要导出

## 7. 当前已实现能力（As-Is 快照）

1. 公共监控：任务列表、任务详情、MTR详情
2. 登录后台：Dashboard、Tasks、Alerts、Webhooks
3. 管理能力：Agents、Releases、Users、Groups、Audit、System Health
4. 支撑能力：任务分配、告警规则、Webhook delivery/retry/test、审计日志

## 8. v1.1 交付优先级

## Phase A（P0）
- 异常优先视图
- 告警到诊断一跳直达
- 首屏价值锚点与模块结果化命名

## Phase B（P1）
- 监控详情结论卡
- 告警规则模板化

## Phase C（P2）
- SLA/复盘输出能力

## 9. 成功指标（建议）

1. 发现到定位时间中位数下降（MTTD->Root Cause Time）
2. 告警事件到有效行动的转化率提升
3. 页面跨跳次数下降（关键路径点击数）
4. 周报/复盘导出使用率提升

## 10. 非目标（v1.1）

- 不在本版本重构后端监控采集架构
- 不在本版本引入复杂自动化编排引擎
- 不在本版本做大规模视觉品牌改版
