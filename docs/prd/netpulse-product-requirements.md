# NetPulse 产品需求文档（PRD）

Last updated: 2026-05-19
Owner: NetPulse Frontend
Status: Active

## 1. 产品背景

NetPulse 是一个面向网络可用性与链路质量的监控平台，目标是通过多地域 Agent 对目标进行持续探测，帮助团队快速回答三个核心问题：

1. 当前网络是否可用？
2. 故障发生在什么范围、什么链路环节？
3. 是否可以基于规则自动告警并联动外部系统？

相较于仅靠单点监控或离线排查，NetPulse 强调：
- 多 Agent 观测视角
- 任务化管理（Task）
- 可视化时序分析（含 MTR）
- 告警与 Webhook 联动

## 2. 产品目标

### 2.1 业务目标

- 降低网络故障发现与定位时延（MTTD/MTTR）
- 建立统一的网络监控与告警运营入口
- 支持团队从“被动排障”转向“主动监控 + 自动通知”

### 2.2 产品能力目标

- 提供公开可访问的监控展示入口（Public Monitoring）
- 提供登录后的任务、Agent、告警、Webhook 管理能力
- 提供管理员级系统治理能力（用户、分组、审计、健康）

## 3. 目标用户

### 3.1 未登录访客（Viewer）

- 关注公开监控看板和任务状态
- 需要快速查看目标可达性与基础链路质量

### 3.2 登录用户（Operator）

- 负责任务维护、监控观察、告警处理
- 需要日常运维操作与状态追踪能力

### 3.3 管理员（Admin）

- 负责 Agent 生命周期、发布升级、权限与治理
- 需要平台管理与可审计操作能力

## 4. 需求价值

### 4.1 对业务的价值

- 提高网络稳定性可见度，减少“故障不可见”时间窗口
- 通过规则告警和 Webhook 触达，缩短故障响应链路
- 降低跨团队沟通成本，形成统一事实来源（single source of truth）

### 4.2 对用户的价值

- Viewer：无需登录即可查看关键监控结果
- Operator：以任务为中心完成监控运维闭环
- Admin：通过统一后台完成资源治理和发布管理

### 4.3 对平台演进的价值

- 前后端 API 已形成较高覆盖，可持续扩展产品面
- OpenSpec + PRD 的变更流程可提升迭代可控性

## 5. 当前已实现功能（As-Is）

## 5.1 认证与访问控制

- 登录 / 注册 / 登出
- Token 刷新链路
- 路由守卫：未登录、登录、管理员权限分层

## 5.2 公共监控（未登录可访问）

- 监控任务列表页
- 监控详情页（按任务）
- MTR 详情页

## 5.3 仪表盘与监控分析

- Dashboard 概览统计
- 健康卡片与监控图表
- 多 Agent 监控数据展示
- MTR 结果查询与明细展示

## 5.4 任务管理（Tasks）

- 任务列表/详情
- 创建任务、更新任务、启停与停用
- 任务与 Agent 分配/解绑

## 5.5 Agent 与发布管理

- Agent 列表与详情
- Agent 创建、更新、停用
- Release 上传、列表、删除、推送更新

## 5.6 告警能力

- 告警规则列表与 CRUD
- 告警事件列表与事件明细查询

## 5.7 Webhook 能力

- Webhook 列表与 CRUD
- 启停状态切换
- Test、Rotate Secret
- Delivery 列表与 Retry

## 5.8 用户与组织治理（Admin）

- 用户列表与禁用
- 修改用户密码
- 分组（Groups）管理（创建/更新/删除）
- 审计日志查看
- 系统健康页

## 6. 登录态能力边界（当前）

### 6.1 未登录可用

- `/monitoring`
- `/monitoring/:taskUuid`
- `/monitoring/:taskUuid/mtr`
- `/login`、`/register`

### 6.2 登录后可用（普通用户）

- `/dashboard`
- `/tasks`、`/tasks/:taskUuid`
- `/alerts`、`/alerts/events`
- `/webhooks`

### 6.3 管理员额外可用

- `/agents`、`/agents/:agentUuid`、`/agents/releases`
- `/users`
- `/groups`
- `/audit`
- `/system/health`

## 7. 当前阶段结论

当前产品已具备“监控采集展示 + 运维管理 + 告警通知 + 管理治理”的核心闭环，能够支撑中小规模网络可观测场景落地。后续优先项应集中在“后端能力前端化补齐”和“运营效率体验优化”上（参见 `docs/prd/todo.md`）。
